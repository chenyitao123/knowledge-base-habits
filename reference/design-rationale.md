# 底层逻辑与认知

> 需要时加载：`read skills/knowledge-base-habits/reference/design-rationale.md`

## 这个库不是帮你整理笔记的

大多数知识管理工具的逻辑是：你有一个想法 → 放进正确的文件夹 → 以后找得到。这是**图书馆思维**——知识是静态的物件，管理就是给它一个位置。

但这个库的设计前提恰好相反：**你的知识不是静态的。** 一个想法今年记下来，明年回头看你会发现它变了——不是你记错了，是你长大了。一个三年前觉得「就是这样」的判断，现在看可能幼稚；一个当时觉得没用的经历，后来成了关键拼图。

所以设计目标不是「整理」，是**帮你「长」出自己的认知体系**。

## 三个反直觉的设计选择

### 第一：按验证次数分层，不按主题分类

传统知识管理问：「这是什么？」→ 放到对应文件夹。

这里问的是**「你验证过几次？」**

```
只经历过一次        → 日记里记一笔
同一类事做过三次以上  → 提炼成方法论（器/）
多个方法论之间浮现模式 → 写成洞察（不器/）
洞察积累到改变了
「你为什么做这件事」  → 更新世界观（所以为人/）
```

每一层不是标签，是你对这件事的**确信程度**。

这是**工程现实主义**——不靠感觉说「我懂了」，靠验证次数。你说你掌握了一个方法，那你做过三次以上吗？你说你有了洞察，那它能解释至少三件已经发生的事吗？

一个想法的位置不是固定的：它在日记里是一个碎片，随着验证次数增加，它会长成方法论，然后可能长成洞察，最终可能改变你的世界观。路径是单向的，但只有验证才能推动。

### 第二：MOC 是阅读路径，不是目录

200+ 篇笔记的文件清单 = 垃圾信息。你在 200 个标题里找不到你要读什么。

所以 MOC 设计成**认知地图**——不是告诉你「这里有什么」，而是告诉你**「如果你想解决 X 问题，读 A→B→C；如果你想理解 Y 框架，读 D→E→F」**。

这是设计意图的根本不同：

| | 传统目录索引 | MOC 认知地图 |
|---|---|---|
| 回答的问题 | 「有什么？」 | 「怎么读？为什么这样读？」 |
| 组织方式 | 字母/时间排序 | 按问题域/认知线索 |
| 对读者的价值 | 碰运气翻找 | 按目的选择路径 |
| 隐藏功能 | 无 | 暴露知识缺口 |

MOC 的隐藏功能：一条阅读路径写到一半发现缺了某篇笔记 → **MOC 本身就暴露了知识缺口**。它不是整理工具，是诊断工具。

### 第三：校准和生长是两条独立的回路

方向锚定不是写一次就锁死的。你需要两个东西：

| | 校准回路 | 生长回路 |
|---|---|---|
| 回答的问题 | 「我偏了吗？」 | 「我变了吗？」 |
| 触发信号 | 行动偏离方向 | 新洞察出现 |
| 方向锚状态 | 固定（需要拉回来） | 本身在进化 |
| 产出 | 恢复计划 | 世界观更新 |
| 晴川角色 | 镜子——呈现偏离事实 | 对话伙伴——整理证据链 |

大多数系统只做第一条（纠偏——「你偏离了，拉回来」）。但你的世界观还是雏形——它需要在学习中**长出来**，不能一直被锁定在一个早期的锚上反复纠偏。

校准说「你说了要做 A，但你最近一直在做 B——要不要调回来？」
生长说「你最近学到的东西，让你对 A 的理解本身变了——要不要更新 A？」

两条回路各自独立触发，各自有不同的产出。但它们的共同前提是：晴川不做裁判，只做镜子/对话伙伴。最终的决定永远属于你。

## 君子不器：三层命名的设计哲学

三层结构不是随便起的名。「器 → 不器 → 所以为人」来自「君子不器」，在知识库设计里每一层都有精准的含义：

### 器/ — 工具层

你学会了一个方法、掌握了一个技能。这层是「器」——具体用途，别人能照着做。

但设计上它**不能**停在这。`verified_count` 字段到 3 之后，系统会推你向上走——提炼流程的 Step 5 明确要求检查「是否浮现了模式 → 启动不器层洞察」。

这就是「不器」的第一层含义：你不会满足于当一个工具人。

### 不器/ — 打破工具边界

当你从多个方法论里看到了模式——当你发现「游戏行为分析的方法竟然也能解释学习拖延」——这就是「不器」的时刻。

工具被超越了，变成了洞察。`contradicts` 矛盾字段在这一层强制存在：每条不器层洞察必须标注与之矛盾的其他笔记。这条规则确保你**不会只看到一面**——洞察不是因为没看到反面才成立，是因为把反面也装进去了才坚固。

### 所以为人/ — 最终指向

洞察积累到改变了「你为什么做这件事」的理解。这是名字本身的含义——工具（器）被超越（不器），最终指向的是「你是什么样的人、你为什么做这些」（所以为人）。

## 底层信念

知识管理的本质不是分类，是用经验验证认知。

你真正知道的不是你能背出来的东西，是你**做过三次以上、看到过模式、它改变了你决策方式**的东西。

两个字段承载了这个信念：

- `verified_count`（器层）——确保你不会停在工具状态，但也确保你不会在验证不足时空谈洞察
- `contradicts`（不器层）——确保你的洞察不是选择性注意的产物，是真正经得起反面审视的
