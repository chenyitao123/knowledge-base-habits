# 经验 → 能力转化流程

> 需要时加载：`read skills/knowledge-base-habits/reference/experience-to-method.md`

## 五步流程

```
Step 1: 记录（自动/指令）
  → 日记中记录：做了什么、结果怎样、什么感受

Step 2: 积累（自动监测）
  → 同一类操作出现 ≥3 次 → 晴川主动标记：✅ 可提炼

Step 3: 提炼（晴川主导）
  → 从 3 次经验中抽取共同步骤、不同结果的归因
  → 写成工具/层笔记：「场景 → 步骤 → 排障 → 底层逻辑」
  → **不强制区分这是「器」（方法）还是「不器」（洞察）——问的是「这次积累了什么能力」**
  🔴 CHECKPOINT：提炼前必须问用户——「你在 X 上积累了 3 次经验，要不要提炼成方法论？」

Step 4: 验证（用户执行）
  → 下次遇到同类场景，按笔记操作
  → 验证通过 → verified_count +1
  → 验证失败 → 更新排障表，标注「已验证失败」

Step 5: 升级（晴川主导 + 用户确认）
  → verified_count ≥ 3 → 升为 evergreen
  → 如果多次经验浮现了超越单个工具的认知模式 → 标记 frontmatter `depth: insight`，同层内提升
```

## 上行链路检查（提炼时同步执行）

提炼完成时，检查人/层的方向锚定和品牌/层的输出计划：
- 新提炼的能力是否支撑了人/层已有的方向理解？
- 如果相关但未引用 → 在人/层对应笔记的 `connects_with` 中追加引用
- 如果是全新能力线，尚无对应的人/层理解 → 标记 `⏳ 待人层连接`
- 新提炼的能力是否可以直接成为品牌层输出的素材？→ 如果相关，在品牌/层笔记中标记 `⏳ 待产出`

此检查确保：能力积累不会停在工具层，它们应该向上支撑自我理解，向外驱动品牌输出。

## 升级触发总览

| 升级 | 条件 | 动作 |
|------|------|------|
| 日记 → 工具/ | 同一类操作 ≥3 次 | 提炼方法论/洞察笔记（不强制区分器/不器） |
| 工具/ → 人/ | 多条方法论/洞察改变了对自己的理解 | 写方向校准笔记，更新 MEMORY.md |
| 人/ → 品牌/ | 对自我的理解有了对外表达的冲动或需求 | 创建/更新品牌层输出（文章、产品、研究） |
