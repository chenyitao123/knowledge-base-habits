# 经验 → 方法论转换流程

> 需要时加载：`read skills/knowledge-base-habits/reference/experience-to-method.md`

## 五步流程

```
Step 1: 记录（自动/指令）
  → 日记中记录：做了什么、结果怎样、什么感受

Step 2: 积累（自动监测）
  → 同一类操作出现 ≥3 次 → 晴川主动标记：✅ 可提炼

Step 3: 提炼（晴川主导）
  → 从 3 次经验中抽取共同步骤、不同结果的归因
  → 写成器/层笔记：「场景 → 步骤 → 排障 → 底层逻辑」
  🔴 CHECKPOINT：提炼前必须问用户——「你在 X 上积累了 3 次经验，要不要提炼成方法论？」

Step 4: 验证（用户执行）
  → 下次遇到同类场景，按笔记操作
  → 验证通过 → verified_count +1
  → 验证失败 → 更新排障表，标注「已验证失败」

Step 5: 升级（晴川主导 + 用户确认）
  → verified_count ≥ 3 → 升为 evergreen
  → 如果多次经验浮现了模式 → 启动 不器/ 层洞察

## 上行链路检查（提炼时同步执行）

提炼完成时，检查本目录的 不器/ 层洞察：
- 它们的 `evidence` 字段是否包含了刚提炼或更新的 器/ 方法论？
- 如果未包含但内容相关 → 在洞察的 `connects_with` 中追加引用
- 如果是全新方法论，尚无对应洞察 → 标记 `⏳ 待不器层连接`

此检查确保：方法论不会被遗忘在器层，它们应该向上支撑洞察。
```

## 升级触发总览

| 升级 | 条件 | 动作 |
|------|------|------|
| 日记 → 器 | 同一类操作 ≥3 次 | 提炼步骤/排障/速查笔记 |
| 日记 → 不器 | 几条日记浮现矛盾或模式 | 写洞察/分析笔记 |
| 不器 → 所以为人 | 洞察改变了对使命的理解 | 写方向校准笔记，更新 MEMORY.md |
