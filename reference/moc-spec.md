# MOC 规范参考

> 需要时加载：`read skills/knowledge-base-habits/reference/moc-spec.md`

## MOC 三层表达

| 层 | MOC 回答 | 组织形式 |
|----|---------|---------|
| 器/ | 「我要解决什么问题？用什么方法？」 | 按问题域分组 |
| 不器/ | 「这条洞察路径通往什么理解？」 | 按认知线索串联 |
| 所以为人/ | 「我为什么做这些？我是什么样的人？」 | 按自我问题组织 |

## MOC 创建与更新

| 触发事件 | 动作 |
|---------|------|
| 目录下笔记 ≥ 3 篇 | **必须建 MOC** |
| 本目录新增笔记 | 更新「状态」行 + 判断是否需要新增阅读路径 |
| 跨目录迁移 ≥ 5 篇 | 重新扫描目录，**重建**阅读路径分组 |
| 删除笔记 | 检查 MOC 中引用 → 移除 |
| 每季度（3/6/9/12月） | 审视 MOC 结构 |

## MOC 模板

```markdown
---
area: [器/不器/所以为人]
status: evergreen
created: YYYY-MM-DD
updated: YYYY-MM-DD
---

# [目录名] · 认知地图

> **这条路径帮你回答**：[一句话]

### 核心概念
- [[关键笔记A]] — 定义了这个领域的基本问题
- ...（3-5 个足矣）

### 📖 阅读路径

### 路径一：[主题名]
> 适合：当你需要 [具体场景] 时

- [[笔记1]] — 干什么用
- [[笔记2]] — 干什么用
→ 读完后你会：[明确的收获]

### 🔗 跨层连接
- 向上 → [[上层洞察/方向校准]]
- 向下 → [[具体实践经验]]
- 横向 → [[同层相关领域MOC]]

### 📊 状态
- 笔记：X 篇 | 已验证：X | 需连接：X
```

## 反例

- ❌ 迁移/新增后不检查 MOC
- ❌ 做成字母排序的文件清单
- ❌ 目录 1-2 篇就建 MOC
- ❌ MOC 没有跨层连接
- ❌ 笔记变了 MOC 不更新
