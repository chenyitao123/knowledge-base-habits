# 世界观生长机制

> 需要时加载：`read skills/knowledge-base-habits/reference/worldview-growth.md`

## 核心理念

世界观校准回路解决「你偏了吗？」— 对比固定方向锚和行动，纠偏。
世界观生长机制解决 **「你变了吗？」**— 新知识进入后，方向锚本身在进化。

雏形世界观 → 学习中吸收新框架 → 方向锚自我升级 → 更新 MEMORY.md

## 三种生长模式

### 模式 A：注入（新洞察挑战了旧认知）

触发条件：一篇新不器/层洞察的 `contradicts` 字段指向了所以为人/层已有的方向锚定笔记。

动作：
1. 晴川主动问：「这个新洞察似乎跟你之前对 [X] 的理解有张力——是之前的理解需要修正，还是两个都成立但需要区分场景？」
2. 用户回答 → 写一篇「世界观更新」笔记放入 所以为人/，标注 `triggered_by: [[新洞察]]`，`replaces: [[旧认知]]`（如果完全替代）或 `extends: [[旧认知]]`（如果叠加）
3. 更新 MEMORY.md 对应部分

### 模式 B：连接（两个孤立的认知突然连上了）

触发条件：两篇所以为人/层笔记之前互不引用，但在新洞察的 `evidence` 或 `connects_with` 中同时出现。

动作：
1. 晴川主动标注：「A 和 B 之前各自独立，但新洞察 [C] 把它们连起来了——它们之间是什么关系？」
2. 用户回答 → 写一篇「世界观合成」笔记，连接 A 和 B

### 模式 C：升级（方向锚本身不够用了）

触发条件：
- 连续 3 次不器/层洞察都在挑战同一个世界观元素
- 用户主动表达「我之前以为的 [X] 好像不太对」
- 遇到一个事件，让旧方向锚显得不够解释力

动作：
1. 晴川整理被挑战的证据链（哪几个洞察、各自指向什么）
2. 问用户：「你的 [X] 这个理解，现在看来需要更新——新的理解是什么？」
3. 用户确认 → 写新版方向锚定笔记，旧版标记 `superseded_by: [[新版]]`

## 与校准的区别

| | 校准 | 生长 |
|------|------|------|
| 方向锚状态 | 固定 | 本身在变 |
| 触发信号 | 行动偏离 | 新洞察出现 |
| 产出 | 恢复计划 | 世界观更新 |
| 问题 | 「你偏了」 | 「你变了」 |
| 晴川角色 | 镜子 | 对话伙伴 + 证据整理 |

## 生长节奏

世界观生长不能追求快。每季度最多 2-3 次生长更新——如果一个月内发生 5 次以上「注入」，可能是认知不稳定而非真实生长。
